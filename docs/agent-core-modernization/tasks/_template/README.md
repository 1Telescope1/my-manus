## Intent

说明本任务解决的问题。引用总任务清单和 SDD，不复制整套总体设计。

## 本任务做了什么

### 一句话说明

> 用一句不依赖代码背景的话说明本任务的本质。

### 为什么需要这个任务

说明改造前的真实问题、对用户或开发的影响，以及为什么不能继续维持原状。

### 核心对象或能力

使用表格或带例子的列表解释核心概念、各自职责和适用场景，不能只列类型名。

### 主要流程

使用文字流程、代码块或必要的图说明输入如何经过关键步骤得到输出，以及重要判断的先后顺序。

### 例子

至少提供一个贴近真实使用场景的例子，说明正常路径；存在重要失败路径时同时给出反例。

### 保护规则和当前边界

说明失败、安全、并发或兼容保护，并明确哪些能力已经可用、哪些尚未接入或属于后续任务。

## Scope

### Out of scope
